# Findings

- 2026-04-21 — The template tree currently mixes three classes of files: fully annotated modular templates, partially annotated modular templates, and aggregate/reference docs with no frontmatter.
- 2026-04-21 — Initial inventory found 42 templates with no frontmatter, 115 missing `title`, 85 missing `type`, and 121 missing `status`.
- 2026-04-21 — The highest-volume first-pass targets are handler families: 26 trigger files, 24 orchestrator files, and 22 operator files each miss `title`, `type`, and `status`.
- 2026-04-21 — Standardizing handlers first reduced the repo-wide debt to 43 missing `title`, 13 missing `type`, and 49 missing `status`, confirming the handler families were the correct first slice.
- 2026-04-21 — The portability gap was not the schema itself; it was that metadata enforcement scope lived only in repo-specific reasoning instead of an explicit policy file.
- 2026-04-21 — Standardizing behaviors next reduced the repo-wide debt to 32 missing `title`, 2 missing `type`, and 38 missing `status`, proving the policy model works on a second legacy frontmatter shape.
- 2026-04-21 — Standardizing guides reduced the repo-wide debt to 32 missing `title`, 1 missing `type`, and 31 missing `status`; the remaining work is now concentrated in a few well-defined families instead of being spread across the whole repo.
- 2026-04-22 — If an active task remains open across days, the prior day must be formally closed during rollover before a new session starts; otherwise `sessions/current` and the work-tracking trail drift out of sync.
- 2026-04-22 — Guard previously treated an uncommitted prior-day active folder as “new today”; checking the tracker `Started` date closes that gap without forcing task folders to rotate every day.
- 2026-04-22 — The matrices family was a clean fourth rollout slice: all eight files already had usable frontmatter, so the work was purely additive (`title` + `status`) plus policy/test activation.
- 2026-04-22 — The registry component family followed the same additive pattern as matrices: non-exempt modules already had stable `type` values, so the rollout mostly required `title` + `status` plus explicit policy exclusions for the aggregate index/report docs.
- 2026-04-22 — Guard and pytest should not be run in parallel when the tests create temporary ACTIVE folders; doing so makes guard see transient fixture directories and report a false multi-folder failure.
- 2026-04-22 — Once engine modules and the two outliers were standardized, the enforced metadata scan reached zero remaining files; the remaining missing keys now belong only to explicitly exempt aggregate or narrative docs.

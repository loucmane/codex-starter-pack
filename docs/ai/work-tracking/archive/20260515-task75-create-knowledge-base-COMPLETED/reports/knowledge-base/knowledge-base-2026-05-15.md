- Use the Markdown runbook for human navigation.
- Use the JSON payload for automation, grep-like discovery, or future MCP installer/index work.
- Run with --query to generate focused packets for a concept, task, command, or workflow.
- Update source documentation at its canonical path; do not edit generated knowledge-base packets as source of truth.

- No source documentation, report, Taskmaster task, session, plan, work-tracking artifact, Serena memory, template, Git state, or external state is mutated beyond requested knowledge-base report artifacts.
- No copy-export tree, duplicate documentation store, database, full-text search index service, crawler daemon, scheduler, notification system, or webhook is created.
- No reader identity, permission model, audit log, usage analytics, or contribution workflow is inferred beyond existing repo-native evidence.

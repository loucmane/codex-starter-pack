# Task 21 CI Fix Evidence

**Date**: 2026-05-07
**Task**: 21 - Implement Template Frontmatter Schema
**PR**: #41

## Failure

GitHub Actions failed after `scripts/codex-guard` began importing `yaml` and `jsonschema`.

Observed failure:

- `ModuleNotFoundError: No module named 'yaml'`

Affected workflows:

- `.github/workflows/codex-guard.yml`
- `.github/workflows/meta-workflow-guard.yml`

## Root Cause

The local environment had project dependencies installed, but GitHub Actions did not install the dependencies declared in `pyproject.toml` before running `scripts/codex-guard`.

## Fix

Both guard workflows now install the `[project].dependencies` list from `pyproject.toml` before running guard commands. This keeps CI aligned with the repo dependency source instead of duplicating dependency names in workflow YAML.

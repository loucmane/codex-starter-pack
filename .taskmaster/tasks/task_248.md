# Task ID: 248

**Title:** Implement First-Class Codex Hook Adapter

**Status:** in-progress

**Dependencies:** 247 ✓

**Priority:** high

**Description:** Add a managed, verified Codex hook adapter that treats canonical apply_patch invocations as first-class Aegis mutation events. The adapter must strictly parse Codex patch syntax, enforce existing Aegis readiness/path/pending/evidence policies, and install `.codex/hooks.json` independently from Claude hook configuration.

**Details:**

Implement this in the existing Python Aegis installer/runtime architecture. Extend `scripts/_aegis_installer.py` where Codex assets, gates, profile metadata, manifests, verification, runtime reload guidance, and package/live asset parity are already managed (`CODEX_REQUIRED_FILES`, `_adapter_assets`, `_gates`, `plan_install`, `install`, `verify`, `runtime_update`, `list_profiles`, `explain_profile`). Add a Codex-only adapter surface alongside the existing Claude `.claude/settings.json` hook model: render and manage `.codex/hooks.json` plus any required `.codex/scripts/*` dispatcher assets, while keeping Claude settings/hooks fully independent. Add these new Codex assets to packaged assets under `aegis_foundation/assets/...` and keep `tests/meta_workflow_guard/test_assets_scripts_parity.py` green by mirroring live/package runtime bytes exactly.

Add a strict canonical apply_patch parser in the shared hook runtime, preferably by extending `.claude/scripts/gate_lib.py` only if it remains a shared Aegis hook library, or by introducing a Codex-specific sibling module that uses the same policy helpers. The parser must accept only the canonical structure: `*** Begin Patch`, one or more operations, and `*** End Patch`; supported operations are `*** Add File: <path>`, `*** Update File: <path>`, `*** Delete File: <path>`, and `*** Move to: <path>` inside a valid update block. Normalize every source and destination path to repo-relative POSIX form before policy checks and event emission. Reject malformed, empty, ambiguous, duplicate, path-traversing, absolute, unsupported, or partially parsed structures fail-closed, including duplicate affected paths and invalid move/update combinations.

Wire Codex hook execution through `.codex/hooks.json` to call the installed `.aegis/bin/aegis hook ...` runtime path where possible, matching the Claude dispatcher pattern in `_render_claude_runtime_dispatcher`. Codex hooks must apply the same Aegis policy stack already enforced for Claude in `gate_lib.py`: readiness state from `.aegis/state/current-work.json`, strict/advisory enforcement from `.aegis/state/enforcement.json`, degraded fail-closed behavior for mutating/unclassified payloads, protected path boundaries (`templates/`, `.codex/`, `.aegis/`, `.claude/`, `CODEX.md`, `CLAUDE.md`, `AGENTS.md`, `scripts/codex-*`, `scripts/template-*`), workflow-owned paths (`sessions/`, `plans/`, `docs/ai/work-tracking/` excluding allowed report evidence), pending tracking from `.aegis/state/pending-tracking.json`, and post-tool evidence behavior.

Emit exactly one atomic Aegis pending/evidence event per accepted apply_patch payload. The event must use handler `codex:apply_patch`, include a deterministic digest of the normalized patch operations/content, choose a primary evidence path deterministically, include the complete sorted affected path set, and record operation metadata for add/update/delete/move. Ensure this integrates with existing pending tracking helpers (`_pending_tracking_events`, `_write_pending_tracking_events`, `record_pending_tracking_event`, `log_work`) so `aegis log --pending-id current` and MCP `aegis.log` can consume Codex-generated events the same way as Claude-generated events.

Update installer planning/adoption rules so `.codex/hooks.json` is managed as a Codex adapter asset with manifest ownership, checksum baselines, manual-review refusal for unsafe adoption, and reload/trust guidance specific to Codex. Preserve the existing safe adoption behavior used by `CLAUDE.md`, `CODEX.md`, and `AGENTS.md`: exact managed content can be upgraded, local project context is preserved in entrypoints, but customized unmanaged hook files require manual review instead of overwrite. Add Codex gates to `_gates()` for hook presence, dispatcher executability, manifest ownership, and runtime verification; make `verify()` fail when required Codex hook paths are missing or drifted. Update `aegis_mcp/server.py` hard-stop/reload guidance so Codex-only installs and multi-agent installs report the correct client reload/trust action without saying Claude reload is required.

Keep existing live/package runtime parity byte-identical. Any new live hook runtime file must be added to `pyproject.toml` package data if needed, mirrored under `aegis_foundation/assets`, included in release distribution tests, and covered by packaged install planning from `tests/meta_workflow_guard/test_aegis_release_distribution.py`. Maintain compatibility with current profile validation in MCP (`primary_agent` values `claude`, `codex`, `gemini`, `multi`, `none`) and make Codex profile/manifest/verification output explicit enough for safe adoption and manual-review refusal.

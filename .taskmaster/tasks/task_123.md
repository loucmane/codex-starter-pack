# Task ID: 123

**Title:** Prove Aegis Release Candidate Global MCP Install on a User-Supplied Existing Project

**Status:** in-progress

**Dependencies:** 122 ✓

**Priority:** medium

**Description:** Build a local Aegis release-candidate artifact and prove it can be registered, started, installed, and used from outside this repository against a copied existing project supplied by the user. The proof must exercise the intended global MCP command path, use Aegis MCP for workflow control-plane operations, use native tools for source edits, and produce durable release evidence before any TestPyPI/PyPI decision.

**Details:**

Codebase analysis: `pyproject.toml` already packages `aegis-foundation` with console scripts `aegis = aegis_foundation.cli:main` and `aegis-mcp-server = aegis_mcp.server:main`, includes `aegis_foundation*`, `aegis_mcp*`, and packaged assets under `aegis_foundation/assets/**`. Package CLI wiring in `aegis_foundation/cli.py` delegates to `scripts/_aegis_installer.py` for `inspect`, `plan-install`, `install`, `status`, `next`, `kickoff`, `log`, `verify`, `closeout`, `certify-release`, and `mcp generate/execute/verify-registration`. Native global registration helpers in `aegis_foundation/mcp_registration.py` render `claude mcp add` and `codex mcp add` commands around `uvx --from <package-spec> aegis-mcp-server --default-target-dir . --transport stdio`, with package, pinned, GitHub, wheel, and source modes. MCP tool/resource/prompt discovery lives in `aegis_mcp/server.py`, where `V1_TOOL_NAMES` includes `aegis.inspect`, `aegis.status`, `aegis.next`, `aegis.plan_install`, `aegis.install`, `aegis.verify`, `aegis.closeout`, `aegis.closeout_ready`, `aegis.kickoff`, `aegis.log`, `aegis.list_profiles`, and `aegis.explain_profile`. The installer already creates `.aegis/state/current-work.json`, `.aegis/state/pending-tracking.json`, `.aegis/bin/aegis`, sessions, plans, work-tracking scaffolds, hooks, reports, strict verification, and closeout reports through `scripts/_aegis_installer.py`. Existing validation lives in `tests/meta_workflow_guard/test_aegis_release_distribution.py`, `test_aegis_native_mcp_registration.py`, `test_aegis_mcp_e2e_targets.py`, `test_aegis_installer.py`, and docs under `docs/aegis/`, especially `distribution.md`, `mcp-client-setup.md`, `release-policy.md`, `release-verification-matrix.md`, and `live-acceptance-matrix.md`.

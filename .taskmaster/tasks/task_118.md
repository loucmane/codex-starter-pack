# Task ID: 118

**Title:** Native Global MCP Bootstrap for Aegis

**Status:** in-progress

**Dependencies:** 117 ✓

**Priority:** medium

**Description:** Make Aegis registerable from any fresh project through native Claude and Codex MCP client commands, with deterministic command generation, optional execution, verification, and documentation for package, pinned, GitHub, local wheel, and source checkout install modes.

**Details:**

Codebase context: Aegis is already packaged in `pyproject.toml` as `aegis-foundation` with console scripts `aegis` and `aegis-mcp-server`; package CLI parsing lives in `aegis_foundation/cli.py`; the repository-local wrapper mirrors Aegis commands in `scripts/codex-task`; deterministic runtime behavior lives in `scripts/_aegis_installer.py`; MCP server startup and discovery surfaces live in `aegis_mcp/server.py` through `AegisMCPConfig`, `V1_TOOL_NAMES`, `RESOURCE_URIS`, `PROMPT_NAMES`, `--default-target-dir`, `--transport`, and `--describe-config`; release docs currently show manual `.mcp.json` / Codex config snippets in `docs/aegis/mcp-client-setup.md` and `docs/aegis/distribution.md`, mirrored into `aegis_foundation/assets/docs/aegis/`; release and MCP coverage already exists in `tests/meta_workflow_guard/test_aegis_release_distribution.py`, `tests/meta_workflow_guard/test_aegis_mcp_server.py`, `tests/meta_workflow_guard/test_aegis_invocation_contract.py`, and `tests/meta_workflow_guard/test_aegis_mcp_e2e_targets.py`. Implement a native MCP registration layer as the default bootstrap path. Add a deterministic registration module, preferably a small package module such as `aegis_foundation/mcp_registration.py` or a shared `scripts/_aegis_mcp_registration.py` if keeping the current script-core pattern is more consistent, with typed client specs for `claude` and `codex`, install source modes for unpinned package (`aegis-foundation`), pinned package (`aegis-foundation==<version>`), GitHub URL or ref, local wheel path, and source checkout path, plus stable JSON output containing client, scope, server name `aegis`, package spec, generated argv, rendered command, target dir, transport, and safety notes. Generate native command argv, not shell fragments, for examples including `claude mcp add --scope user aegis -- uvx --from aegis-foundation aegis-mcp-server --default-target-dir . --transport stdio`, `claude mcp add --scope project aegis -- uvx --from aegis-foundation aegis-mcp-server --default-target-dir . --transport stdio`, and `codex mcp add aegis -- uvx --from aegis-foundation aegis-mcp-server --default-target-dir . --transport stdio`. Preserve the existing MCP server command shape from `aegis_mcp/server.py`; only the client registration wrapper changes. Add package CLI subcommands in `aegis_foundation/cli.py`, and matching repo-local support in `scripts/codex-task`, for generate, execute, and verify flows, for example `aegis mcp generate-registration`, `aegis mcp execute-registration`, and `aegis mcp verify-registration` or an equivalent argparse shape that matches existing hyphenated command style. Generate mode must be read-only and print JSON by default, with an optional command rendering suitable for docs. Execute mode must locate the requested native client with `shutil.which`, run `subprocess.run` with the exact argv list, capture stdout/stderr/returncode into JSON, and never use `shell=True`. Verify mode must use native client inspection commands for Claude and Codex, parse their stdout/stderr robustly enough to recognize the `aegis` server, the `uvx --from ... aegis-mcp-server` command, `--default-target-dir`, and `--transport stdio`, and return structured pass/fail checks; add an optional stdio probe that launches the generated MCP server command and verifies discovery of `aegis.*` tools using the existing MCP client patterns from release tests. Missing client binaries must fail with a clear JSON error such as `status: missing_client` and include the command that would have been run. Make direct config-file writes an explicit fallback only: add a flag such as `--write-config-fallback` or `--fallback-config-file` for legacy/manual flows, keep it off by default, and ensure docs state native client registration is preferred. Ensure generated commands work without cloning `/home/loucmane/codex`: default to package assets, keep source checkout mode explicit, and verify `aegis-mcp-server --describe-config` reports `asset_origin: package` unless a source path was requested. Update docs in both `docs/aegis/` and packaged assets for global Claude, project Claude, Codex, pinned package, GitHub URL/ref, local wheel, and source checkout flows; replace manual `.mcp.json` as the primary path with native `claude mcp add` / `codex mcp add`, while retaining manual config snippets only under a fallback section. Update the release verification matrix to include native client registration, verification parsing, missing-client behavior, and fresh-project bootstrap acceptance. Acceptance: from a fresh directory with no checkout, a user can run the generated native client registration command globally or per-project, the native client can discover the Aegis MCP server and `aegis.*` tools, and the registered tool path can install packaged runtime assets and then run `aegis.kickoff`, `aegis.log`, `aegis.verify`, and `aegis.closeout` without requiring this repository to exist locally.

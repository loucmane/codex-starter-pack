**Details:**

Codebase analysis shows Aegis is packaged as `aegis-foundation` in `pyproject.toml`, with CLI entry points in `aegis_foundation/cli.py` and MCP tools in `aegis_mcp/server.py`. Core workflow behavior lives in `scripts/_aegis_installer.py` and must stay mirrored into `aegis_foundation/assets/scripts/_aegis_installer.py`; existing acceptance helpers live in `tests/meta_workflow_guard/aegis_acceptance_assertions.py`, fixture coverage in `tests/meta_workflow_guard/test_aegis_installer_fixtures.py`, MCP contracts in `tests/meta_workflow_guard/test_aegis_mcp_server.py`, and prior normal-language acceptance scope in `plans/2026-05-29-task130-aegis-normal-language-workflow-acceptance.md`. Extend this system for a Taskmaster-backed Claude MCP target: create or reuse a fresh fixture project that has `.taskmaster/` and installed Aegis, then run a live or scripted Claude acceptance path from a normal-language request. The required path is: discover Taskmaster work via `task-master next` and `task-master show <id>` or equivalent MCP calls; start Aegis with the explicit Taskmaster numeric id using `aegis.kickoff apply=true` rather than `aegis.start`; create the branch, session, plan, and work-tracking scaffold using that Taskmaster id; log scope with `aegis.log` before source edits; implement only through native agent tools; consume pending tracking and log implementation evidence through Aegis; run project-specific verification; run `aegis.verify strict=true`; repair failures only through Aegis tools such as `aegis.repair`, `aegis.handoff_repair`, `aegis.log`, and re-verification; run `aegis.closeout_ready` or dry-run closeout, then `aegis.closeout`; run `aegis.doctor`; only after Aegis closeout and doctor pass, mark the Taskmaster task done with `task-master set-status --id=<id> --status=done` and targeted generated-file refresh. Harden guidance in `scripts/_aegis_installer.py`, `aegis_mcp/server.py`, packaged `aegis_foundation/assets/CODEX.md`, and `docs/aegis/*` so installed Claude guidance prefers Taskmaster discovery and explicit `aegis.kickoff` whenever `.taskmaster/` has an available numeric task. Preserve the local `aegis.start` path for projects without Taskmaster, but make the distinction explicit in `aegis.next`, MCP prompts such as `aegis.start_task` and `aegis.closeout_task`, and public docs. Add guardrails and assertions that reject the known failure modes: huge checklist prompts, synthetic handler labels, direct edits under `.aegis/` workflow state, hidden Serena requirements, use of local Aegis task ids when a Taskmaster task is available, Taskmaster `done` before Aegis closeout and doctor, and broad `task-master generate` when a targeted refresh is sufficient.

# Feedback Collection Reports

Feedback collection packets are static JSON/Markdown artifacts generated by:

```bash
python3 scripts/codex-task feedback collection-plan \
  --report-file reports/feedback-collection/latest.json \
  --runbook-file reports/feedback-collection/latest.md
```

Use this report when an operator needs a repository-native intake and triage model for feedback collected during foundation adoption or task review.

The packet includes:

- current workflow, Taskmaster, Git, and Serena evidence snapshots
- feedback evidence domains and refresh commands
- intake schema fields
- categories and manual severity/sentiment labels
- routing matrix and response workflow states
- metrics checklist
- archive guidance and manual next steps
- explicit non-goals

This report is planning-only. It does not create hosted forms, API endpoints, dashboards, sentiment automation, notifications, tickets, external archives, response delivery systems, Taskmaster updates from feedback rows, or external integrations.

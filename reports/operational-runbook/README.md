# Operational Runbook Reports

This directory is reserved for reusable operational runbook packets generated by:

```bash
python3 scripts/codex-task operations runbook \
  --label <label> \
  --report-file reports/operational-runbook/<label>.json \
  --runbook-file reports/operational-runbook/<label>.md
```

The command composes existing portable foundation helpers into one operator-facing JSON and Markdown packet:

- daily session start, implementation logging, and closeout procedures;
- weekly, monthly, quarterly, and yearly static maintenance cadences;
- incident, recovery, emergency, rollback, and change-advisory routing;
- troubleshooting matrix for common workflow failures;
- role-based escalation guidance;
- validation checklist mapped to existing evidence commands.

The packet is static evidence. It does not install schedulers, start daemons, send notifications, create tickets, update dashboards, deploy code, execute rollback, mutate Taskmaster state, or contact external incident systems.

Task-specific operational runbook evidence should still be stored under the active work-tracking reports folder:

```text
docs/ai/work-tracking/active/<YYYYMMDD-task-slug-ACTIVE>/reports/operational-runbook/
```

# Production Verification Packets

This directory is reserved for reusable static production verification packets generated by:

```bash
python3 scripts/codex-task deployment verification \
  --report-file reports/production-verification/latest.json \
  --runbook-file reports/production-verification/latest.md
```

The command composes existing portable foundation evidence into one final verification review packet:

- workflow and Taskmaster health;
- final validation suite evidence;
- security audit and compliance limitation evidence;
- static performance benchmark evidence;
- static cost-governance evidence;
- emergency, recovery, rollback, and operational runbook posture;
- static monitoring coverage;
- final documentation evidence;
- stakeholder sign-off readiness;
- Task 80 production transition readiness evidence.

The packet is static and evidence-backed. It does not deploy code, run live security scans, certify compliance, benchmark production traffic, query billing systems, execute disaster recovery, activate monitoring, send stakeholder communications, publish releases, switch traffic, perform rollback, or contact external systems.

Reusable packets may be written here when a project owner wants an ongoing verification artifact. Task-specific production verification evidence should still be stored under the active work-tracking reports folder:

```text
docs/ai/work-tracking/active/<YYYYMMDD-task-slug-ACTIVE>/reports/production-verification/
```

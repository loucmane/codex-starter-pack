# Production Deployment Readiness Packets

This directory is reserved for reusable static production transition readiness packets generated by:

```bash
python3 scripts/codex-task deployment readiness \
  --report-file reports/production-deployment/latest.json \
  --runbook-file reports/production-deployment/latest.md
```

The command composes existing portable foundation evidence into one release/BAU transition review packet:

- workflow and Taskmaster health;
- final validation suite evidence;
- final documentation map evidence;
- maintenance and BAU readiness;
- post-migration monitoring cadence;
- stakeholder communication readiness;
- celebration/readout planning;
- cleanup/archive posture;
- rollout and adoption guidance;
- historical production-deployment requirement mapping.

The packet is static and evidence-backed. It does not deploy code, enable services, start monitoring, install schedulers, run auto-fix, mutate cleanup targets, send communications, schedule events, split traffic, publish releases, or contact external systems.

Reusable packets may be written here when a project owner wants an ongoing readiness artifact. Task-specific production deployment evidence should still be stored under the active work-tracking reports folder:

```text
docs/ai/work-tracking/active/<YYYYMMDD-task-slug-ACTIVE>/reports/production-deployment/
```

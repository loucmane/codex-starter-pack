# Continuous Improvement Reports

This directory stores reusable static continuous-improvement review packets generated by:

```bash
python3 scripts/codex-task enhancement continuous-improvement \
  --label <label> \
  --report-file reports/continuous-improvement/<label>.json \
  --runbook-file reports/continuous-improvement/<label>.md
```

The command composes existing portable-foundation evidence into one non-destructive improvement loop review:

- feedback intake and triage from the feedback collection packet;
- enhancement roadmap and innovation pipeline evidence from the Phase 5 enhancement packet;
- metric-driven selection from success metrics and template quality evidence;
- experiment and change validation from A/B planning, change advisory, and final validation evidence;
- learning and communication from post-mortems, stakeholder reporting, and the knowledge base;
- maintenance and operating cadence from the maintenance plan and operational runbook.

## Outputs

- `<label>.json` - machine-readable loop stages, domain status, evidence paths, review queue, refresh commands, guidance, and non-goals.
- `<label>.md` - Markdown review packet for handoff and planning review.

Task-specific continuous-improvement evidence should still be stored under the active work-tracking reports folder:

```text
docs/ai/work-tracking/active/<YYYYMMDD-task-slug-ACTIVE>/reports/continuous-improvement/
```

## Boundaries

This packet is static evidence. It does not create a suggestion portal, experimentation runtime, feature flag service, live dashboard, scheduler, notification system, ticket, PR, approval, owner assignment, analytics backend, or external integration. Accepted improvement work still needs a scoped Taskmaster task, session, plan, work-tracking folder, tests, and handoff evidence.

# Cleanup Automation Reports

Cleanup automation packets are static JSON/Markdown artifacts generated by:

```bash
python3 scripts/codex-task cleanup plan \
  --report-file reports/cleanup-automation/latest.json \
  --runbook-file reports/cleanup-automation/latest.md
```

Use this report when an operator needs a repository-native cleanup review plan before touching legacy artifacts, generated scanner outputs, backup directories, deprecated templates, or workflow state.

The packet includes:

- current workflow, Taskmaster, Git, and Serena evidence snapshots
- cleanup evidence domains and refresh commands
- cleanup candidate classes and risk labels
- approval gates
- dry-run checks
- backup and rollback guidance
- metrics checklist
- manual notification guidance
- explicit non-goals

This report is planning-only. It does not install schedulers, delete files, move files, archive files, create backups, run rollback, send notifications, enforce retention, mutate Taskmaster from packet rows, or contact external systems.

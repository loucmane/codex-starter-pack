# Long-term Maintenance Reports

This directory is reserved for reusable long-term maintenance packets generated by:

```bash
python3 scripts/codex-task maintenance plan \
  --label <label> \
  --report-file reports/maintenance/<label>.json \
  --runbook-file reports/maintenance/<label>.md
```

The command composes existing portable foundation evidence into one static maintenance control packet:

- workflow health from current session, plan, work-tracking, guard, and Taskmaster state;
- operational cadence from the operational runbook helper;
- post-migration monitoring evidence and recurring review cadence;
- performance baseline evidence;
- template quality scorecard evidence;
- cleanup planning evidence;
- security audit and manual patch-review evidence;
- dependency inventory and manual update-review guidance.

The packet is static evidence. It does not install schedulers, start daemons, send notifications, create tickets, apply patches, update dependencies, create dashboards, mutate templates, or contact external services.

Task-specific maintenance evidence should still be stored under the active work-tracking reports folder:

```text
docs/ai/work-tracking/active/<YYYYMMDD-task-slug-ACTIVE>/reports/long-term-maintenance/
```

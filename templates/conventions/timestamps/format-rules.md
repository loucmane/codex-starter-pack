# Timestamp Format Rules

## Convention
All timestamps must be generated from system commands, never typed manually, and follow consistent formats for their context.

## Critical Rule

### 🚫 NEVER Type Timestamps Manually
```bash
# ❌ WRONG - Typed from memory
"2025-07-30 14:30 CEST"  # 30% error rate when typed

# ✅ CORRECT - Generated from command
date "+%Y-%m-%d %H:%M %Z"  # Always accurate
```

### Why This Matters
- Human timestamp typing has 30% error rate
- Wrong timestamps break chronological ordering
- Incorrect dates cause session corruption
- Time zones often forgotten or wrong

## Standard Formats

### Full Timestamp (ISO-like)
```bash
# Command
date "+%Y-%m-%d %H:%M %Z"

# Output
2025-07-30 14:30 CEST

# Usage
- sessions/ entries
- Progress logs
- Handoff documents
```

### Date Only
```bash
# Command
date +%Y-%m-%d

# Output
2025-07-30

# Usage
- TRACKER.md headers
- Daily logs
- Changelog entries
```

### Folder Names
```bash
# Command
date +%Y%m%d

# Output
20250730

# Usage
- Work tracking folders
- Archive directories
- Backup folders
```

### Time Only
```bash
# Command
date '+%H:%M'

# Output
14:30

# Usage
- Progress log entries
- Session timestamps
- Quick time notes
```

### ISO 8601 Full
```bash
# Command
date -Iseconds

# Output
2025-07-30T14:30:45+02:00

# Usage
- APIs
- Database entries
- Machine-readable logs
```

### Unix Timestamp
```bash
# Command
date +%s

# Output
1690722630

# Usage
- Performance metrics
- Cache keys
- Unique identifiers
```

## Context-Specific Formats

### sessions/ Entries
```markdown
## Session: 2025-07-30 14:30 CEST
```
Command: `date "+%Y-%m-%d %H:%M %Z"`

### Memory File Names
```
session_2025-07-30_template_migration.md
```
Command: `date +%Y-%m-%d`

### Work Folders
```
20250730-feature-name-ACTIVE
```
Command: `date +%Y%m%d`

### Git Commits
```bash
gac "feat: add feature at $(date '+%Y-%m-%d %H:%M')"
```

### Log Entries
```
[2025-07-30T14:30:45.123Z] Log message
```
Command: `date -u +%Y-%m-%dT%H:%M:%S.%3NZ`

## Time Zone Handling

### Local Time with Zone
```bash
# Shows local time with timezone
date "+%Y-%m-%d %H:%M %Z"
# Output: 2025-07-30 14:30 CEST
```

### UTC Time
```bash
# Forces UTC output
date -u "+%Y-%m-%d %H:%M UTC"
# Output: 2025-07-30 12:30 UTC
```

### Specific Timezone
```bash
# Show time in specific zone
TZ='America/New_York' date "+%Y-%m-%d %H:%M %Z"
# Output: 2025-07-30 08:30 EDT
```

## Common Commands Reference

### Quick Copy-Paste
```bash
# Full timestamp
date "+%Y-%m-%d %H:%M %Z"

# Date only
date +%Y-%m-%d

# Folder date
date +%Y%m%d

# Time only
date '+%H:%M'

# ISO 8601
date -Iseconds

# Unix timestamp
date +%s

# Milliseconds since epoch
date +%s%3N
```

## Validation Examples

### ✅ Correct Usage
```markdown
## Session: 2025-07-30 14:30 CEST
**Generated by**: `date "+%Y-%m-%d %H:%M %Z"`

### Progress Log
- **14:30** - Started work (from `date '+%H:%M'`)
- **15:45** - Checkpoint reached (from `date '+%H:%M'`)

**Work Folder**: 20250730-migration-ACTIVE
**Created with**: `mkdir "$(date +%Y%m%d)-migration-ACTIVE"`
```

### ❌ Wrong Usage
```markdown
## Session: 2025-01-30 14:30 CEST  # Wrong month typed
**Error**: Typed manually, should use command

### Progress Log  
- **14:30** - Started work  # Might be wrong time
- **15:45** - Next entry    # Probably guessed

**Work Folder**: 20250130-migration-ACTIVE  # Wrong date
**Error**: Typed date instead of using command
```

## Special Cases

### Relative Dates
```bash
# Yesterday
date -d "yesterday" +%Y-%m-%d

# Last week
date -d "1 week ago" +%Y-%m-%d

# Next month
date -d "next month" +%Y-%m-%d

# Specific date
date -d "2025-07-30" +%A  # Day of week
```

### Duration Calculations
```bash
# Time since start
START=$(date +%s)
# ... work happens ...
END=$(date +%s)
DURATION=$((END - START))
echo "Took $DURATION seconds"
```

### File Timestamps
```bash
# File modification time
stat -c %y filename.txt

# File creation time (birth)
stat -c %w filename.txt

# Last access time
stat -c %x filename.txt
```

## Platform Differences

### Linux/WSL
```bash
date "+%Y-%m-%d %H:%M %Z"
```

### macOS
```bash
# Same as Linux but different stat command
stat -f %Sm -t "%Y-%m-%d %H:%M" filename.txt
```

### Windows (PowerShell)
```powershell
Get-Date -Format "yyyy-MM-dd HH:mm"
```

## Anti-Patterns

### Never Do This
```markdown
# ❌ Typing timestamps
"2025-07-30 14:30"  # Typed from memory

# ❌ Guessing times
"around 2pm"  # Too vague
"sometime yesterday"  # Not specific

# ❌ Wrong format
"07/30/2025"  # US format in international context
"30.07.2025"  # European format in US context

# ❌ Missing timezone
"2025-07-30 14:30"  # Which timezone?
```

## Rationale

### Why These Rules

1. **Accuracy**: Commands always give correct time
2. **Consistency**: Same format everywhere
3. **Sorting**: YYYY-MM-DD sorts chronologically
4. **Timezone Clarity**: Always include zone info
5. **Automation**: Commands can be scripted

### Benefits
- **No Errors**: Eliminates timestamp mistakes
- **Traceability**: Accurate audit trails
- **Internationalization**: ISO format works globally
- **Debugging**: Correct times aid troubleshooting
- **Automation**: Consistent format enables tooling
